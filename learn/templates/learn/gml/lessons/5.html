Lógica De Programação

Lógica e condições

Vamos agora analizar mais algumas outras coisas, aprofundando um pouco mais… Podemos ainda usar o exemplo de ir no mercado:

  • Levante;
  • Saia de casa;
  • Chegue na calçada;
  • Olhe para os dois lados da rua;
  • Se estiver passando carros:
  • Espere;
  • Se não:
  • Atravesse;
  • Entre no mercado;
  • Pegue o refrigerante;
  • Vá até o caixa;
  • Pague;
  • Fim;

Podemos verificar que, neste código, temos condições, consegue identificá-las? As condições são:

Condição 1:

se estiver passando carros: -> Condição


Faça algo (Espere;) -> Comando que deve ser executado se a condição 1 for confirmada.


Condição 2:

Se não (estiver passando carros): -> Condição contrária a primeira.


Faça outra coisa (Atravesse;) -> Comando que deve ser executado se a condição 2 for confimada.

Só será confirmada a condição 2 se a 1 não for, e então esta condição 1 será ignorada, e vice-versa. Uma condição anula a outra, então no momento que você chega nessa parte, supondo que não esteja passando carros, você não terá que tomar a primeira atitude: Espere;, e ela será totalmente ignorada, afinal, existe a outra condição que diz: se não:, neste caso você fará apenas a atitude da condição: se não:, e a condição: se estiver passando carros: com seu comando interno: Espere; será ignorada. Isso pode parecer ridículo e você pode pensar: “isso é óbvio”, mas eu estou dando este exemplo para te explicar que a lógica da programação é bem parecida com a lógica humana, a lógica muitas vezes é confundida com algo óbvio, e isso também se aplica na programação.

Veja que neste trecho do código tem a condição, e a atitude de cada condição está mais “para dentro”, você percebe? Esse “mais pra dentro” chama-se indentação, geralmente na programação usamos a indentação ou para tornar o código mais legível ou então para delimitar um bloco de código para uma determinada atitude ou condição específica para que ele funcione corretamente, isso varia de lingugem para linguagem. Repare também como por exemplo a forma para dizer que uma linha chegou ao seu fim, como você pode reparar neste código de ir ao mercado que estamos usando como exemplo, cada final de linha tem no final um “;” eu usei isso para se tornar mais legível na explicação indicando onde termina cada passo do código… Mas a respeito do uso desse tipo de ferramenta na programação ser necessária ou não, como eu já disse, varia de linguagem para linguagem, para entrar nesse assunto, vamos falar um pouco de sintaxe...
