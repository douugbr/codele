Lógica De Programação

Input e Output

Vamos entender os conceitos de input e output, de maneira literal, podemos entender:

  • input = entrada
  • output = saída

Basicamente, input é a inserção de dados, e output é a saída dos dados… Vou usar um exemplo mais próximo do dia a dia, vamos supor que você tenha um amigo que seja desenhista, vamos chamá-lo de Gustavo. Gustavo é um artista que ama desenhar… suponhamos que Gustavo não consegue desenhar sem ouvir música, isso porque, é através da música que ele tira inspiração para suas artes, então basicamente, o dispositivo Gustavo recebe um input que é a música, e quando ele recebe esse input, ele libera o output que é o desenho, então:

  • input = música/inspiração
  • output = desenho

Na programação, o input nada mais é do que receber algum dado, e o output seria liberar dados. Quando você escreve o código, você está fazendo um input, você está inserindo dados no código, e quando você roda o programa, você libera seu output, que é o que o seu programa realiza quando você coloca ele pra “rodar”. Ou então quando você faz um programa que interaja com o usuário, por exemplo, você faz um programa que pergunte ao usuário o seu nome, aí o quando o usuário digita seu nome e aperta Enter, ele está fazendo um input em seu programa, ele está inserindo o dado que é o seu nome, depois que ele faz isso, seu programa mostra uma mensagem dando boas vindas ao usuário, chamando pelo seu nome, que no caso é o output. Vamos imaginar que já fizemos esse programa, e agora colocamos ele pra rodar, você não precisa ver agora a estrutura do código, apenas repare no programa rodando:

Código que você escreveu e salvou (Input):

  • Escreva("Qual é o seu nome?: ")
  • Escreva("Olá {Variável "Nome"}! Como vai?")


Programa rodando no terminal(Output):

  • >>> Qual é o seu nome? : Gustavo -> Input que o usuário fez colocando seu nome.


  • Output:

  • >>> Olá Gustavo! como vai? -> Output que libera a mensagem de boas vindas.

Este é o terminal... Aquela famosa "tela preta" onde o programa está sendo executado depois que você colocou ele para rodar
