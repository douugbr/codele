Introdução à plataforma

Criando o primeiro jogo

Na ultima aula, aprendemos como criar um projeto no Game Maker Studio 2, e sobre suas principais ferramentas (Objetos, Sprites, Scripts, entre outros). Na Aula de hoje, iremos começar a interagir com estas ferramentas, para entender melhor seu funcionamento de forma prática, e criarmos nosso primeiro mini game na plataforma.


Primeiros passos

Para começarmos nosso projeto, em primeira mão é necessário um objeto para que possamos interagir. Para criar um objeto, clique na barra “Objetos” (Que fica no lado direito da tela, na aba “Resources”) com o botão direito do mouse, e clique em “Create Object”, ou então, clique nas teclas “ALT+O” no teclado para criar um objeto rapidamente. Agora que o objeto foi criado, surgiu uma nova janela para ele em nosso Workspace, e nela podemos manipular alguns atributos de nosso objeto, como se ele será visível na room aos olhos do jogador, com a opção “Visible”, e entre outras customizações, que entenderemos melhor futuramente.

Renomeie o objeto para “obj_player”, e em seguida na janela “Event” que estará ao lado da janela do objeto, clique em “Add Event” para abrir as opções de eventos que podemos utilizar para programar nosso objeto.


Conhecendo os Eventos

Eventos, são a base para orientação dos objetos dentro da plataforma, de forma resumida, eles que ditam quando coisas vão acontecer, por exemplo, quando o player conseguirá atirar, pular, quando determinada ação irá acontecer, determinam quando o player irá colidir com paredes e objetos, entre diversas outras coisas.

Os principais eventos que iremos utilizar durante nosso curso, serão: Create, Mouse, Destroy, Other, Alarm, Draw, Step e Collision.

  • Create – O evento create é responsável por executar o código dentro dele, no momento em que o objeto é criado na Room.

  • Mouse – Evento utilizado para usar o mouse em determinada situação, seja para executar comandos, controlar o personagem, e afins.

  • Destroy – Quando o código dentro do evento “Destroy” for executado, o objeto será destruído.

  • Other – Alguns outros eventos, como por exemplo: Game Start (Executa código quando o jogo começa), Game End (Executa código quando jogo termina), Room Start (Executa código quando a room é iniciada).

  • Alarm – Cria um evento de tempo, onde é criado um alarme, que quando chega a 0, executa o código presente no evento.

  • Draw – Usado para desenhar imagens na tela quando determinado código for executado. Apenas neste evento os códigos com “draw” no inicio funcionam.

  • Step – É um dos evento mais importantes do jogo, senão o mais importante. O Step executa o código dentro dele a cada frame do jogo continuamente.

  • Collision – Executa um código quando o objeto colide com um objeto determinado.


Criando e movimentando o jogador

Em primeira mão, criaremos apenas um cubo para ser nosso personagem. Depois de ter criado o objeto do personagem, é hora de criar uma sprite para ele. Ao criar a sprite, clique em “Edit image”, e com o balde de tinta do editor de imagem, pinte o quadrado com a cor que desejar. Depois de pintado, volte para o Workspace. Agora, chegou a hora de centralizar a sprite de nosso personagem, para fazer isso, clique na caixa onde há escrito “Top Left” ( se encontra acima da imagem da sprite no centro), e mude esta opção para “Middle Centre”, para centralizar as coordenadas x e y da sprite.

Agora, de volta ao objeto player, iremos selecionar a sprite que acabamos de criar. Na janela “Obj_Player” no Workspace, abaixo do nome do objeto, estará a parte onde se muda a sprite do mesmo. Clique na caixa onde está marcado “No sprite”, e selecione a Sprite que criamos.

Agora que criamos nosso personagem, chegou a hora de começarmos a dar os primeiros passos para fazer ele se movimentar. Vamos criar um evento de “Create” para nosso personagem, para que possamos atribuir alguns valores e características para o jogador.

Com o evento “Create” criado, iremos criar uma variável chamada “velocidade”, e seu valor será igual a 5. O código ficará parecido com isto:

Velocidade = 5

A velocidade do personagem é um valor de suma importância para começarmos a programar a movimentação do objeto.

Depois de terminarmos no evento create, hora de criar um novo evento, o “Step”. No momento em que clicarmos no evento step, ele nos mostrará 3 opções de Step: Step, Begin Step, e End Step. Porém, em primeira mão iremos apenas trabalhar com o Step. Depois de criado, iremos escrever um código para fazer o personagem andar para a direita, depois, iremos explicar passo a passo como o código funciona. Para fazer o personagem andar para a direta, o código necessário será:

  • if keyboard_check(vk_right) = true {
  • x+=velocidade
  • }


EXPLICANDO O CÓDIGO:

Para entendermos esse código, é necessário compreender o que cada parte dele significa, e tentarmos ler ele como se fosse uma frase.

  • If = se

  • kbeyboard_check() = determinada tecla for pressionada

  • vk_right = seta direita do teclado

  • = true = for verdadeira

  • {

  • X = posição do objeto na room

  • += velocidade = adição de valor na posição x do personagem, ou seja, a posição dele mais 5 (que é a velocidade)

  • }

Se fossemos ler esse código como uma frase, se pareceria como: se a seta direta do teclado for pressionada, o x do personagem irá crescer em 5 (velocidade). E como o evento step verificada o código dentro dele a cada segundo, enquanto a seta direita for pressionada, o objeto continuará avançando para a direita.

Agora que terminamos a programação do personagem, vamos testa-la colocando nosso personagem numa room, e iniciando o jogo. Vá na aba “Rooms” no game Maker, e provavelmente já irá ter uma chamada “Room 0”, caso não haja, crie uma e abra-a. Para adicionar um objeto a uma room, basta arrasta-lo da aba de objetos para dentro da room. Como o objeto já dentro da room, hora de iniciar o jogo, apertando a tecla F5.
